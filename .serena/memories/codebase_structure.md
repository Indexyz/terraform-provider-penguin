# Codebase structure

- `main.go`: provider server entrypoint (`providerserver.Serve`), sets `ServeOpts.Address` (still `registry.terraform.io/hashicorp/scaffolding`).
- `internal/provider/`: Terraform Plugin Framework provider implementation and example components:
  - `provider.go`: provider schema/configure + lists of Resources/DataSources/Functions/Actions/EphemeralResources.
  - `example_resource.go`, `example_data_source.go`, `example_function.go`, `example_action.go`, `example_ephemeral_resource.go`: template examples.
  - `*_test.go`: unit/acceptance-style tests using `terraform-plugin-testing` patterns.
- `examples/`: Terraform configuration snippets used by docs generation.
- `docs/`: generated provider/resource/data-source/function/action docs (generated by `tfplugindocs`).
- `tools/`: `go generate` directives for license headers, `terraform fmt`, and docs generation (`tools/tools.go`).
- `penguin/`: Penguin service API docs + TS type definitions (not integrated with the Go provider code).
- `.golangci.yml`: lint configuration.
- `GNUmakefile`: common dev targets.
- `.goreleaser.yml`: release build configuration.
