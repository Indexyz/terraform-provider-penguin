# Style & conventions

## Go style
- Standard Go formatting via `gofmt -s` (see `GNUmakefile: fmt`).
- Lint via `golangci-lint` with an explicit allowlist (see `.golangci.yml`).

## Terraform Plugin Framework patterns (as used in `internal/provider/*`)
- Provider/resources/data sources implement framework interfaces and assert them via `var _ ... = &Type{}`.
- Configuration/state models are structs with `tfsdk:"..."` tags and use `types.*` values.
- Typical flow:
  - Read config/plan/state via `req.Config.Get`, `req.Plan.Get`, `req.State.Get`.
  - Accumulate diagnostics via `resp.Diagnostics.Append(...)` and early-return on `HasError()`.
  - Use `Configure` methods to accept `req.ProviderData` (type assert, add diagnostic errors on mismatch).
- Logging uses `tflog`.

## Naming
- Exported types for framework objects (`ScaffoldingProvider`, `ExampleResource`, etc.) and unexported fields for internal state (e.g. `client *http.Client`).

## Generated content
- `docs/` is generated by `terraform-plugin-docs`; avoid manual edits.
- License header automation exists via `copywrite` (`.copywrite.hcl`).
